### 2. Key Backend Concepts

- **Database Design:**  
  Learned normalization, entity-relationship modeling, and indexing strategies to optimize queries and maintain data integrity.

- **RESTful API Development:**  
  Built scalable, versioned, and well-documented REST APIs following HTTP methods and status code conventions.

- **GraphQL API Integration:**  
  Implemented flexible queries and mutations for efficient client-server communication.

- **Asynchronous Programming:**  
  Applied async views and Celery workers to handle long-running tasks without blocking main processes.

  Used Redis to cache expensive database queries and improve performance in high-traffic applications.

- **Authentication & Authorization:**  
  Integrated JWT-based authentication and role-based permissions to secure endpoints.

- **System Design:**  
  Understood scalability, load balancing, microservices, and deployment architectures for production-ready systems.

## üß© Best Practices & Takeaways

- Follow **Django design patterns** (Models-Views-Serializers) for clean architecture.
- Maintain **modular code** and use environment variables for configuration.
- Write **comprehensive documentation** for every API endpoint.
- Apply **unit and integration testing** before deployment.
- Adopt **Git branching strategies** for clean version control.
- Use **Continuous Integration (CI)** pipelines to automate testing and linting.
- Prioritize **security and scalability** in every project.
